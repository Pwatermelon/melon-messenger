# Ротация ключа шифрования сообщений (at-rest)

## Важно: не каждые 7 дней

`MESSAGE_AT_REST_KEY` защищает **копию базы** (дамп ScyllaDB на диске). Это не сессионный токен.

- **Менять ключ каждые 7 дней не нужно** и не принято: старые сообщения всё равно останутся на старом ключе, пока вы их не перешифруете.
- Достаточно **одного сильного ключа** + хранение секретов в `.env` / GitHub Secrets.
- Ротация имеет смысл при **подозрении на утечку**, смене сервера, уходе человека с доступа к секретам — **по событию**, а не по календарю.

Если ключ сменить **без** поддержки старых — сообщения не расшифруются (раньше API отдавал «мусор»).

## Как устроено сейчас

- Новые сообщения: `enc:v2:<id_ключа>:...`
- Старые `enc:v1:` — читаются перебором всех ключей из кольца
- В `.env` можно держать **несколько ключей**; активный — для записи

```env
MESSAGE_AT_REST_ACTIVE_KEY_ID=2
MESSAGE_AT_REST_KEY=<новый_ключ>
MESSAGE_AT_REST_KEY_1=<старый_ключ>
MESSAGE_AT_REST_KEY_2=<новый_ключ>
```

`MESSAGE_AT_REST_KEY` должен совпадать с `MESSAGE_AT_REST_KEY_<ACTIVE_ID>`.

## Процедура ротации (без потери данных)

1. Сгенерировать новый ключ:
   ```bash
   openssl rand -base64 32
   ```
2. В `.env` / `PROD_ENV_FILE`:
   - Оставить старый как `MESSAGE_AT_REST_KEY_1=...`
   - Добавить новый: `MESSAGE_AT_REST_KEY_2=...` и `MESSAGE_AT_REST_KEY=...` (тот же)
   - `MESSAGE_AT_REST_ACTIVE_KEY_ID=2`
3. Перезапустить API — **новые** сообщения пишутся новым ключом, **старые** читаются.
4. Перешифровать архив (опционально, но желательно):
   ```bash
   cd apps/api
   bun run db:reencrypt-messages --dry-run   # посмотреть счётчик
   bun run db:reencrypt-messages             # применить
   ```
5. Когда все сообщения на ключе `2` — можно убрать `MESSAGE_AT_REST_KEY_1` из env.

## Что ротировать отдельно

| Секрет | Что сломается при смене | Как менять |
|--------|-------------------------|------------|
| `MESSAGE_AT_REST_KEY` | Старые сообщения в Scylla | Кольцо ключей + `db:reencrypt-messages` |
| `JWT_SECRET` | Все сессии разлогинятся | Просто сменить, пользователи войдут снова |
| `MINIO_ROOT_PASSWORD` | Доступ к медиа | Менять пароль MinIO + обновить env |

## Автоматическая ротация по расписанию

Не рекомендуется без автоматической перешифровки и бэкапа. Если понадобится — лучше cron на сервере:

1. Бэкап Postgres + Scylla
2. Новый ключ в env
3. `db:reencrypt-messages`
4. Рестарт API

Интервал — **месяцы**, не дни.
